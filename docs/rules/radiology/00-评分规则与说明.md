# 影像报告书写规则 · 评分规则与说明

> 本目录用于汇总影像报告**书写规范**与 ASR 输出的**纠错映射**：单位归一、数值×单位写法、缩写大小写、标点、分级评分表达等。每一条都会被批量导入到 `correction_rules` 表，作为转写后处理与报告校对的依据。

## 1. 收录范围
- 影像（X 线 / CT / MRI / 超声 / DSA / 核医学 / PET 等）报告中**确实会出现**的书写规则。
- 包含：度量单位与符号、数值格式、解剖方位、分级量表（BI-RADS / TI-RADS / LI-RADS / PI-RADS / Lung-RADS / TNM / RECIST / Fazekas / Pfirrmann / Modic / Bosniak / O-RADS / Lugano …）、缩写大小写、标点规范、ASR 高频误识别替换。
- 不包含：临床用药剂量、护理流程、行政文书等与影像无关的规则。

## 2. 表格字段

每张表使用统一的 9 列：

| 字段 | 说明 |
| --- | --- |
| `规则类型` | 高层分组标签，例：单位归一、乘号统一、分级写法。规则导入时落入 `conflict_group` 字段，便于冲突检测与回滚。 |
| `错误模式 (pattern)` | ASR/手输出现的常见错误片段。匹配方式为 `literal` 时按字面替换，`regex` 时按 Go regexp 语法。 |
| `正确写法 (replacement)` | 替换为符合影像报告规范的目标文本。Regex 支持 `$1`、`$2` 反向引用。 |
| `匹配方式` | `literal` / `regex` / `number_normalize`（数值归一）。 |
| `优先级` | 整数 1-1000，数值越小越先执行；同一冲突组内冲突时取最小优先级。建议：句法/结构 < 单位 < 缩写 < 拼写。 |
| `冲突组` | 同一类规则用同一冲突组字符串，避免顺序错乱时相互覆盖。 |
| `启用` | `是` / `否`。默认 `是`；标 `否` 的规则会导入但 enabled=false。 |
| `示例` | 触发场景，便于校对。 |
| `备注` | 适用上下文限制、白名单、注意事项等。 |

## 3. 优先级建议（数值越小越先）

| 区间 | 用途 |
| --- | --- |
| 1-29 | 句子级修正、特殊上下文（如「左／右侧」误读）。 |
| 30-59 | 分级量表与缩写大小写（BI-RADS、TNM、Lung-RADS 等）。 |
| 60-89 | 数字-单位归一（毫米→mm、毫米汞柱→mmHg、乘号统一）。 |
| 90-149 | 通用文本替换（标点、口语化词）。 |
| 150+ | 兜底与低置信替换。 |

## 4. 冲突组命名规范

- `unit-normalize`：所有度量单位的中→英标准化（mm/cm/mL/Hu/mmHg/bpm 等）。
- `dimension`：尺寸表达（乘号、× 与 x 的统一、3D 量化）。
- `direction`：解剖方位与左右侧。
- `grading-*`：每张分级量表一个组，如 `grading-birads`、`grading-tirads`、`grading-tnm`、`grading-lirads`、`grading-pirads`、`grading-lungrads`、`grading-orads`、`grading-fazekas`、`grading-pfirrmann`、`grading-modic`、`grading-bosniak`、`grading-recist`、`grading-lugano`、`grading-mtici`、`grading-timi`。
- `abbreviation`：英文缩写大小写归一（MRI/CT/DSA/MRA/MRV/PET-CT 等）。
- `punctuation`：标点统一（中英文逗号、括号、冒号）。
- `vital-sign`：心率、血压、呼吸、体温写法。
- `lab-value`：实验室值与对比剂剂量。

## 6. 与术语库的关系

- **术语库（terms）**：词与词的标准写法和别名（如「胼胝体」对应错误变体「骈祉体/便支体」）。
- **规则库（rules）**：模式与模式的替换（如 `\d+\s*毫米` 替换为 `$1mm`、`BI-?RADS` 大写归一）。
- 两者均挂在 `term_dicts` 下；规则统一挂在 domain=`radiology-rules` 的「影像通用规则」字典；术语挂在各科室专属字典。
