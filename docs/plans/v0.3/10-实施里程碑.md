## 8. 风险登记

| 风险 | 触发条件 | 缓解 |
|---|---|---|
| 鉴权中间件性能回归 | OpenAuth 增加 5 ms+ 延迟 | 引入 token 缓存（10 s） |
| 调用日志写入阻塞主链路 | 高 QPS 下 DB 写入慢 | 异步队列 + 批量入库；每 100 ms 或 100 条 flush |
| Skill 回调地址被恶意指向内部资源 | SSRF | 注册时禁止 RFC1918 + 元数据 IP；white-list 校验 |
| 客户拒绝迁移到新接口 | 长期使用兼容层 | 通过运营 + 计费策略推动；保留兼容层但不再扩展能力 |
| 工作流可见性与现有 OwnerType 冲突 | 数据回填出错 | 写迁移脚本 + 回归测试覆盖三类 owner |
| 兼容接口被恶意公网调用消耗 ASR / GPU 资源 | 无鉴权导致滥用 | 部署侧 nginx 上层做 IP 白名单 / 限流；监控 `legacy-access.log` 异常 IP；不在应用层加 token |
| Legacy 流量被误算入新平台指标 | 数据污染 | 中间件分流 + 编译期隔离 + 验收清单中"跨链路串扰"用例 |
