1. 识别范围标注：普通话、中文、英文（美）


项目	说明
修改范围	产品规格书、概要设计、产品能力接口、ASR language 默认值
后端	1. 产品能力接口返回当前版本支持的语言列表，用于能力说明
2. asr-api 仍支持调用方显式传入 language，但默认值为 auto
3. 调用外部 ASR 服务时，language 为空或 auto 不透传，保留中英文和数字混合识别能力
前端	1. 实时识别页和批量转写页不强制选择语种，不默认携带 zh-CN
2. 产品能力中的语种列表仅作为规格/能力展示，不作为运行时限制
桌面端	1. 设置窗不提供强制识别语种配置
2. 录音识别请求默认不携带 zh-CN，避免破坏混合识别
数据库	transcription_tasks、meetings 表 language 字段默认 auto，记录显式传入时的语种

4. Win7 / Win10 / Win11 客户端兼容性


项目	说明
排查结论	已采用双桌面端方案交付：Windows 10 / Windows 11 由 `desktop/`（Tauri 2）产出主推荐包，复用系统 WebView2；Windows 7 SP1 由新增的 `desktop-electron/`（Electron 22.3.27）产出独立兼容包，不再依赖 WebView2 固定版运行时，也不再需要 Win7 可加载的 Rust/CRT 构建链路。两个包在发布脚本中同时构建，公共下载页按文件名中的 `_win10_` / `_win7_` 标识分组展示；发布验证需分别覆盖 Win7 VM 与 Win10/11 宝环境。
修改范围	桌面端 Tauri 构建配置、安装包打包、启动兼容性代码、下载页兼容性说明
Tauri 配置	1. 已确认 release 入口包含 windows_subsystem = "windows"
2. Windows 交叉打包仅产出 NSIS；WebView2 安装模式使用 fixedRuntime，固定到 WebView2 109.0.1518.78 x64 的预解压目录
3. Win7 构建使用 Rust 1.77.2 + x86_64-pc-windows-msvc，并通过 crt-static 避免 VCRUNTIME/UCRT app-local 依赖
兼容代码	1. 前端构建目标保持不高于 Chrome 109（当前 windows 目标 chrome105）
2. WebView2 权限、全局热键、托盘、剪贴板注入需在 Win7 / Win10 / Win11 分别启动验证
3. 如 Win7 上 WebView2 109 不支持现有能力，需在当前客户端内降级对应能力或补齐 Win7 专用实现，不把运行时安装责任转给客户
安装包	1. 打包脚本硬检查固定版 WebView2 109 目录，缺失时直接失败
2. NSIS 安装脚本不再检测或引导安装系统级 WebView2；运行时由应用安装包内置
备选方案	若 Tauri + WebView2 109 在 Win7 实机验证中出现不可绕过的能力缺口，再新增 Win7 专用客户端框架。优先评估 WPF/.NET Framework 4.8 或 Qt 5 原生客户端；Electron/CEF 只能使用 Chromium 109 或更早版本，安全维护风险更高。

修改优先级建议

P0 高	5, 6, 7, 8	涉及核心功能行为变更，影响用户使用

P1 中	1, 2, 4	新增功能或兼容性改进
P1 中	10	批量导入是运维效率提升，优先级较高
