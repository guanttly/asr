# 巨鲨语音助手 v0.3 — 开放平台规划

> 文档版本：v0.3 草案 1
> 起草日期：2026-04-28
> 状态：规划中（Plan 阶段）

v0.3 的核心主题是：把巨鲨语音助手从"自带前端的私有化部署产品"，扩展成一个**对外提供能力的开放平台**——三方系统通过授权拿到的 token 调用平台能力，平台同时保留对旧接口的兼容直到调用方升级完毕。

## 关键决策（速读）

- **鉴权方案（仅新平台）**：三方系统先在管理端注册自己的应用信息并勾选能力 → 平台签发 `app_id + app_secret`，调用接口时附带 Bearer Token（基于 secret 派生）。新平台的未授权或越权调用一律拒绝。
- **对外能力清单（v0.3 首批）**：
  1. 语音转文字（同步上传 / 异步任务 / 实时流）
  2. 会议纪要（音频→纪要 / 文本→纪要）
  3. 语音指令 Skill（三方注册 skill → 平台识别意图 → 回调三方）
- **工作流附加**：所有能力调用都允许通过 `workflow_id` 指定工作流；不传则使用平台为该应用配置的默认工作流。
- **兼容接口**：`old http.py` 暴露的 9 个接口在 `/api/legacy/*`（同时保留旧 `/api/*` 路径直连）下完整保留，**完全不做鉴权**——存量调用方已经集成、无法回头改造，因此即便存在鉴权"漏洞"也按现状放过；通过路由 / 模块 / 数据 / 日志的功能隔离避免污染新平台。只在内部文档说明，不对外开放规格，待调用方迁移后整体下线。
- **接口文档**：开放平台 API 提供完整的参数表和示例（含 cURL / SDK 片段）；兼容接口不再补充任何对外说明。

## 命名约定

- 文中"应用 / 三方应用 / 调用方"指对接平台的外部系统，对应数据库实体 `OpenApp`。
- "能力（capability）"指平台对外暴露的一类逻辑功能，如 `asr.recognize`、`meeting.summary`、`skill.register`。
- "Skill"专指三方系统注册到平台、用于语音指令回调的能力描述。
- 路径前缀：开放平台 = `/openapi/v1/...`；兼容接口 = `/api/legacy/...`。
