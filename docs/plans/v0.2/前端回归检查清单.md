# 前端回归检查清单

## 1. 导航与布局
- 打开系统后，侧边栏一级菜单显示为：总览、应用、工作流、术语库、系统管理。
- 应用下可见：实时语音识别、批量转写、会议纪要。
- 工作流下可见：工作流管理、应用配置。
- 术语库下可见：术语库管理、纠错规则。
- 折叠侧边栏后，一级和二级菜单图标不变形。
- 折叠侧边栏后，悬停图标能看到 Tooltip。
- 顶栏标题区显示统一 SVG 图标，不再出现临时占位文本。

## 2. 应用配置页
- 从工作流二级菜单进入应用配置页，可看到实时语音识别、批量转写、会议纪要三张配置卡片。
- 每张卡片都能正常加载对应类型工作流列表。
- 选择默认工作流后，刷新页面配置仍保留。
- 清空默认工作流后，状态提示变为“未配置默认工作流”。
- 如果本地保存了失效工作流 ID，页面会显示不可用提示。

## 3. 实时语音识别
- 未配置默认工作流时，页面提示“保存但不触发后处理”。
- 已配置默认工作流时，页面展示正确的默认工作流预览。
- 停止录音后，请求会携带默认工作流 ID。
- 实时任务保存成功后，结果区能显示本次任务实际绑定的工作流。
- 若历史任务绑定的是已失效工作流，页面会显示升级/修复提示。

## 4. 批量转写
- 上传本地音频和提交 URL 两个入口都不再出现页面内工作流选择器。
- 两个入口都会展示当前默认工作流文案和跳转到应用配置的入口。
- 已配置默认工作流时，创建任务和上传文件都会携带对应 workflow_id。
- 未配置默认工作流时，仍可提交任务，但仅执行 ASR。
- 任务详情中，历史任务绑定了失效工作流时会显示明确提示。

## 5. 会议纪要
- 新建会议页不再允许直接选择工作流，只展示默认工作流状态和预览。
- 已配置默认工作流时，新建会议请求会携带对应 workflow_id。
- 会议详情页不再允许页面内切换工作流，只能去应用配置页统一调整。
- 会议详情页点击“生成摘要”时，优先使用应用配置中的默认工作流。
- 如果会议详情继承的是旧绑定工作流且默认配置为空，页面能解释当前实际行为。

## 6. 工作流与术语库
- 工作流管理入口独立存在，不再混在应用入口下。
- 术语库管理和纠错规则都能从菜单直接进入。
- 术语页和规则页切换时，菜单选中状态正确。

## 7. 构建与代码质量
- 执行 `pnpm build` 成功。
- 执行 `pnpm lint` 成功。
- 仓库中不存在未接路由的旧页面残留，例如旧版实时转写页。
