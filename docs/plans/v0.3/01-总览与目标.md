# 01 — v0.3 总览与目标

## 1. 版本定位

v0.3 在 v0.2 工作流编排能力的基础上，新增**对外开放平台**形态，使巨鲨语音助手可以脱离自带前端独立作为能力中台被三方系统集成。
v0.3 不替代现有的 Web / 桌面端，而是把"管理 + 工作流 + 引擎"封装成对外稳定 API，并补齐：

- 三方应用注册与凭证管理；
- 调用方与权限隔离；
- 接口规格与 SDK 片段；
- 与旧版 HTTP 接口（`docs/old http.py`）的过渡兼容。

## 2. 业务目标（与 v0.3 KPI 对齐）

| 编号 | 目标 | 衡量指标 |
|---|---|---|
| OBJ-1 | 三方系统可在 1 个工作日内完成接入 | 从"提交注册"到"首次成功调用"的中位耗时 ≤ 1 个工作日 |
| OBJ-2 | 平台保持对旧接口完全兼容 | `old http.py` 列出的 9 个接口在新版本下行为等价、回归用例 100% 通过 |
| OBJ-3 | 新平台越权调用 0 漏出 | 仅针对 `/openapi/v1/*`：未鉴权 / 越权能力 / 凭证泄漏 / 重放均按规格拒绝；兼容接口不在此约束内 |
| OBJ-4 | 单租户调用平均开销可观测 | Dashboard 增加按 `app_id` 维度的调用量、错误率、平均时延 |
| OBJ-5 | Skill 模式可用 | 至少 1 个内部联调用例：注册 skill → 唤醒词命中 → 调用回调 → 三方拿到结构化 payload |

## 3. 功能范围

### 3.1 In Scope（必须交付）

1. **平台管理后台扩展**
   - "三方应用"管理页：创建、编辑、禁用、能力勾选、Token 重置。
   - "Skill 管理"页：查看 / 调试三方注册的 skill。
   - "调用日志"页：按 `app_id` / 能力 / 状态码筛选最近 7 天调用。
2. **开放 API（`/openapi/v1`）**
   - 语音转文字：同步、异步任务、流式会话。
   - 会议纪要：音频→纪要、文本→纪要、模板列表。
   - Skill：注册、注销、查询；语音意图回调机制。
3. **兼容 API（`/api/legacy` 与旧 `/api/*` 直连）**
   - 完整复刻 `old http.py` 的 9 个接口，**完全不做鉴权**（与新平台功能 / 路由 / 数据隔离），保持响应字段不变。
4. **工作流附加**
   - 所有能力调用支持 `workflow_id` 传入；不传走该应用配置的默认工作流。
5. **可观测性**
   - 每次开放 API / 兼容 API 调用写一条结构化日志（app_id、capability、latency、status）。
   - Dashboard 增加按应用维度的指标卡。
6. **接口文档**
   - 开放 API 文档（含参数表、示例、错误码、SDK 片段）放入 `docs/openapi/`。
   - 兼容接口**不**在 `docs/openapi/` 中暴露规格，仅在内部 `docs/plans/v0.3/08-...` 留映射表。

### 3.2 Out of Scope（v0.3 不做）

- 多租户级别的资源配额隔离与计费（v0.4）。
- OAuth2 / OIDC 登录（v0.4，TODO-02）。
- 公网开放 / 多机房水平扩展（v0.5+，TODO-05）。
- 桌面端对接开放平台（桌面端继续走管理 JWT，不切换到三方 token）。
- 单 Skill 可定义复杂多轮对话（v0.3 仅支持"意图命中即回调"的一次性调用）。

## 4. 与现有版本承诺的关系

- 产品规格书 §3 中的版本矩阵不变，开放平台是**额外能力**，不影响标准版 / 高级版判断逻辑。
- 高级版独有的 `meeting_summary` / `voice_intent` / `speaker_diarize` 节点，在开放 API 上对**未购买高级版的部署**继续以 403 形式拒绝（按 `product.edition` 判定）。
- v0.3 不动数据保留与清理策略（产品规格书 §7.3），但调用日志加入 `runtime/openapi-logs/` 单独清理（默认 30 天）。

## 5. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| 旧接口字段语义与新接口冲突 | 兼容接口失真 | 统一在适配层做字段映射，新接口用 `snake_case`，旧接口保留原字段名（即便重复） |
| 三方 token 泄漏 | 越权调用 / 计量错误 | 仅在创建时返回明文，平台只存哈希；提供"重置 secret"按钮；登录管理后台才能查看 |
| Skill 回调 URL 不可达 | 指令链路断裂 | 注册时探活 + 回调失败重试（指数退避 3 次） |
| 调用日志体量膨胀 | 磁盘 / 数据库压力 | 行存仅留索引字段，正文（请求 / 响应体）写入文件；过期清理 |
| 桌面端 / Web / 兼容接口 / 新平台四套鉴权混淆 | 鉴权混乱、误把兼容流量计入新平台 | 中间件按路径前缀分流：`/api/admin`、`/api/asr`、`/api/meetings`、`/ws/events` 走 JWT；`/openapi/v1` 走 OpenAuth；`/api/legacy/*` 与旧 `/api/upload` 等老路径**不挂任何鉴权中间件**，直接进 legacy 适配层 |
| 兼容接口被恶意公网调用 | 引擎资源被滥用 | 部署侧用 nginx 上层做 IP 白名单 / 限流；不在应用层加 token |

## 6. 非目标但需注意的事项

- 文档面向**对接方**而非最终用户：示例都假设有一个已注册的 `app_id`。
- 兼容接口的"不写文档"是商业策略：避免新接入方继续走旧路径；但内部 plans 必须留下完整映射表，以便后续下线评估。
