## 1 概述

### 1.1 编写目的

为了明确巨鲨语音助手项目的软件需求、安排项目规划与进度、方便日后组织软件开发与测试，特撰写本文档。

1. 本文档是巨鲨语音助手需求规格说明书，供设计人员使用，作为系统设计的依据。
2. 作为巨鲨语音助手项目验收标准之一。
3. 作为该软件日后维护的参考资料之一。

### 1.2 文档范围

本文档作为巨鲨语音助手项目的软件系统规格说明书，使用对象主要包括：

- 项目需求人员
- 软件开发人员
- 项目主管/项目经理
- 测试验收人员

本文档覆盖Web管理端和桌面客户端当前已实现的页面与交互。测试时应以本文档描述的当前实现为准；未在页面中提供的功能不得作为本轮验收项。

### 1.3 术语定义

| 术语 | 定义 |
| ---- | ---- |
| ASR | Automatic Speech Recognition，自动语音识别，将语音信号转换为文本的技术 |
| VAD | Voice Activity Detection，语音活动检测，识别语音信号中是否包含人声 |
| 声纹 | Voiceprint，说话人的生物特征，用于识别说话人身份 |
| 工作流 | 可编排的有序节点序列，对识别结果做后处理 |
| RAG | Retrieval-Augmented Generation，检索增强生成框架 |
| CER | Character Error Rate，字错误率，衡量语音识别准确度的指标 |
| 产品能力 | 后端返回的功能开关，用于控制会议纪要、声纹库、语音控制等高级能力是否可见可用 |
| 匿名登录 | 桌面客户端使用机器码向后台换取JWT的登录方式 |
| 悬浮球 | 桌面客户端常驻的小型录音控制窗口 |

### 1.4 通用约束定义

#### 1.4.1 名称约束（中文及字符输入约束）

对输入的字段，由【中文，a-z，A-Z，0-9，_】字符组成，长度范围为约束条件。

#### 1.4.2 文本约束（文本输入约束）

对输入的字段，由【中文、英文字母、数字、下划线、空格、常用标点符号】字符组成，但不允许以空白字符开头或结尾，也不允许出现连续的空白字符，长度范围 0-N。

#### 1.4.3 数值约束

数值输入字段，需符合最小值和最大值范围约束。

## 2 系统说明

### 2.1 背景介绍

巨鲨语音助手是一套面向局域网私有化部署的端到端语音识别与处理平台，主要解决以下问题：

- 医疗场景下的语音转写需求，支持医学术语准确识别
- 会议纪要自动生成，提高办公效率
- 实时语音识别与后处理工作流编排
- 私有化部署，数据安全可控

- 技术路线：采用Qwen-ASR语音识别引擎，支持工作流可视化编排、术语纠错、说话人分离、LLM智能纠错等功能。

### 2.4 运行环境

| | |
| ---- | ---------------------------------------------------------------------- |
| 硬件环境 | 最低配置：<br>CPU：8核<br>内存：16G<br>存储：200G SSD<br>推荐配置：<br>CPU：16核<br>内存：32G<br>存储：500G SSD |
| 软件环境 | 最低配置：<br>操作系统：Ubuntu 22<br>Docker：≥20.10<br>Docker Compose：≥2.20.0 |
| 算力资源 | 最低：RTX 3090 同等算力<br>推荐：A10/A100 |

### 2.5 条件和限制

1. 支持局域网私有化部署，无需公网连接；
2. 支持标准版和高级版两种授权模式；
3. 实时识别需HTTPS安全上下文；
4. 需配置外部ASR引擎服务；
5. 会议纪要、声纹库、语音控制等能力受产品能力开关控制；未开通时Web端隐藏入口或路由返回看板，桌面端隐藏对应场景和Tab；
6. 桌面客户端默认通过机器码匿名登录，服务地址、令牌、设备别名、识别参数和热键配置保存在本机。

### 3.2 实时语音识别

#### 3.2.1 录音控制

- **用户场景**：
  用户点击开始录音，系统采集麦克风音频并进行实时转写。

- **输入/前置条件**：
  1. 浏览器支持麦克风权限；
  2. HTTPS安全上下文；

- **需求说明**：
  1. 支持开始、暂停、继续、停止录音；
  2. 本地分句后调用短句识别接口；
  3. 本地采集约16k音频并按约300ms音频块进行VAD切句；
  4. 录音结束后自动保存历史任务。

**【约束条件】**

| 序号 | 约束条件 | 条件描述 |
| --- | ---- | ------------------- |
| 1 | 功能 | 支持录音控制（开始/暂停/继续/停止） |
| 2 | 功能 | 本地分句识别，不维持流式会话 |
| 3 | 功能 | 录音结束后自动上传整段音频并保存 |
| 4 | 交互 | 录音状态实时显示（空闲/录音中） |
| 5 | 约束 | 需HTTPS安全上下文 |
| 6 | 功能 | 支持基于应用配置绑定的实时工作流进行即时后处理 |
| 7 | 功能 | 支持模拟片段触发页面输出，用于联调展示 |

**【异常处理】**

| 序号 | 异常说明 | 处理方案 |
| --- | ------ | ---------- |

| 3 | 短句识别失败 | 保留页面状态并提示识别失败原因，不生成该句输出 |

- **补充说明**：无

| 序号 | 异常说明 | 处理方案 |

- **用户场景**：

| 序号 | 约束条件 | 条件描述 |

| 3 | 注入 | 自动注入可在设置中开启或关闭，默认开启 |

**【异常处理】**

| 序号 | 异常说明 | 处理方案 |
| --- | ------ | ---------- |
| 1 | 短句识别失败 | 记录错误并继续处理后续短句 |
| 2 | 实时工作流失败 | 保留原始文本或提示错误，不中断录音 |
| 3 | 自动注入失败 | 结果仍保存在历史中，用户可手动复制 |

- **补充说明**：报告模式用于普通实时转写，不生成会议纪要。

#### 3.18.10 语音控制

- **用户场景**：
  用户通过唤醒词或快捷键进入指令模式，说出指令后切换报告模式或会议模式。

- **输入/前置条件**：
  1. 产品能力已开通语音控制；
  2. 应用配置已绑定语音控制工作流；
  3. 语音控制运行参数为启用。

- **需求说明**：
  1. 普通语音片段先执行唤醒识别；
  2. 命中唤醒后进入命令模式；
  3. 命令模式下的识别文本只用于指令分类，不写入转写历史或注入；
  4. 支持切换到报告模式；
  5. 会议能力开通时支持切换到会议模式；
  6. 支持通过热键直接进入或退出指令模式。

**【约束条件】**

| 序号 | 约束条件 | 条件描述 |
| --- | ---- | ------------------- |
| 1 | 工作流 | 指令识别调用当前账号绑定的语音控制工作流 |
| 2 | 超时 | 命令等待时间来自应用配置，桌面端最低按2秒处理 |
| 3 | 失败 | 连续3次分类失败后退出命令模式，避免持续吞掉语音 |
| 4 | 场景 | 切换场景后刷新工作流绑定，下一句使用最新绑定 |
| 5 | 吞吐 | 未命中唤醒词的文本按普通转写文本处理 |

**【异常处理】**

| 序号 | 异常说明 | 处理方案 |
| --- | ------ | ---------- |
| 1 | 未绑定语音控制工作流 | 不进入有效指令识别，提示或记录失败 |
| 2 | 指令分类超时 | 退出命令模式并恢复原场景 |
| 3 | 未开通语音控制能力 | 隐藏语音控制配置和相关热键动作 |

- **补充说明**：当前语音控制注册表包含报告模式切换和会议模式切换两类意图。

#### 3.18.11 桌面识别参数与调试日志

- **用户场景**：
  用户在设置窗调整本地VAD参数，或开启调试日志协助定位录音、网络、快捷键问题。

- **输入/前置条件**：
  1. 设置窗连接设置浮层已打开。

- **需求说明**：
  1. 支持配置是否保留标点；
  2. 支持配置最小触发阈值、底噪倍数、静音块数、有效语音块数、单块峰值倍数；
  3. 支持请求麦克风权限；
  4. 支持开启或关闭调试日志；
  5. 参数变更保存在本机并影响后续录音。

**【约束条件】**

| 序号 | 约束条件 | 条件描述 |
| --- | ---- | ------------------- |
| 1 | 默认值 | 默认不保留标点，最小阈值0.018，底噪倍数2.8，静音块数4，有效语音块数2，单块峰值倍数1.45 |
| 2 | 存储 | 识别参数保存到本地设置，不写入服务端 |
| 3 | 生效 | 参数对后续录音生效，不回改已保存历史 |
| 4 | 日志 | 调试日志用于本机问题诊断，不作为业务数据展示 |

**【异常处理】**

| 序号 | 异常说明 | 处理方案 |
| --- | ------ | ---------- |
| 1 | 参数不合法 | 使用默认值或限制范围内的值兜底 |
| 2 | 麦克风权限请求失败 | 显示失败原因并保留未授权状态 |

- **补充说明**：Web实时识别和桌面实时识别使用相同思路的本地VAD参数，但分别存储在各自客户端环境中。
