# 12 — 旧客户端兼容接口测试方案

## 1. 验证目标

以 [docs/old http.py](../../old%20http.py) 为基准，验证新版后端在 `legacy.enabled=true` 时完整保留旧客户端可见行为：旧路径免鉴权、旧请求字段可用、旧响应字段可读、`audio/to_summary` 的同步与 `callback` 异步链路都能工作。

本地联调时 ASR 下游固定使用：`http://192.168.40.221:11001`。

## 2. 本地启动配置

在 `backend` 目录启动服务时覆盖 ASR 地址：

```bash
ASR_SERVICES_ASR=http://192.168.40.221:11001 go run ./cmd/asr-api
ASR_SERVICES_ASR=http://192.168.40.221:11001 go run ./cmd/nlp-api
ASR_SERVICES_ASR=http://192.168.40.221:11001 go run ./cmd/gateway
```

默认端口：gateway `10010`、asr-api `10011`、nlp-api `10013`。旧客户端优先访问 gateway：`http://127.0.0.1:10010`。

## 3. 自动化覆盖

核心兼容行为由 Go 测试锁定：

```bash
cd backend
go test ./internal/interfaces/api -run 'TestLegacy'
go test ./internal/interfaces/api
```

当前自动化用例覆盖：

| 用例 | 覆盖点 |
|---|---|
| `TestLegacyRecognizeMatchesOldHTTPShapeAndForwardsOptions` | `/api/recognize` 保留旧顶层 `filename/language/result/timestamp`；同时保留 `data`；`language/use_itn/hotwords` 透传到 ASR |
| `TestLegacyRecognizeVADReturnsOldSegmentsShape` | `/api/recognize/vad` 固定 VAD，返回旧版 `use_vad_segmentation=true` 与 `result.segments` |
| `TestLegacyAudioToSummaryCallbackUsesOldContract` | `/api/audio/to_summary` 接受旧 `callback` 字段、立即返回任务、完成后 POST 旧 envelope，且不附带 OpenAPI HMAC 签名 |
| `TestLegacyAudioToSummaryRejectsInvalidCallback` | 非 HTTP/HTTPS callback 在创建任务前返回 400 |

## 4. 接口矩阵

| 旧路径 | 别名路径 | 验证内容 |
|---|---|---|
| `GET /api/health` | `GET /api/legacy/health` | HTTP 200；`status=healthy`；`service=JushaAsr HTTP Server` |
| `POST /api/upload` | `POST /api/legacy/upload` | multipart `file`；`language/use_itn/use_vad_segmentation/hotwords` 可用；旧顶层 `result` 可读 |
| `POST /api/recognize` | `POST /api/legacy/recognize` | multipart `file`；固定非 VAD；旧顶层 `filename/language/result/timestamp` 可读 |
| `POST /api/recognize/vad` | `POST /api/legacy/recognize/vad` | multipart `file`；固定 VAD；`result.segments` 存在 |
| `POST /api/audio/to_summary` | `POST /api/legacy/audio/to_summary` | multipart `audio_file`；同步返回 `asr_result/llm_processing/processing_info`；异步 `callback` 返回 `task_id` 后触发 POST |
| `GET /api/task/{task_id}` | `GET /api/legacy/task/{task_id}` | 已创建任务可查询；不存在任务返回旧 `success=false/message=任务不存在或已完成` |
| `POST /api/meeting/summary` | `POST /api/legacy/meeting/summary` | JSON `text/template/variables`；返回旧 envelope |
| `POST /api/text/correct` | `POST /api/legacy/text/correct` | JSON `text/context/enable_speaker_identification`；返回旧 envelope |
| `GET /api/templates` | `GET /api/legacy/templates` | 返回 `templates/default_template` |

## 5. 手工联调步骤

1. 启动 callback 接收端，监听本机任意端口并记录请求 body。
2. 使用一段短 wav/mp3 文件依次请求 `/api/recognize`、`/api/recognize/vad`、`/api/audio/to_summary`。
3. `/api/audio/to_summary` 同步模式不传 `callback`，等待返回完整 `data`。
4. 异步模式传 `callback=http://127.0.0.1:<port>/callback`，确认立即响应包含 `task_id/callback_url/created_at`，随后 callback 服务收到 `success=true` 的 JSON body。
5. 用返回的 `task_id` 请求 `/api/task/{task_id}`，确认状态字段可读。
6. 对每条旧路径重复请求 `/api/legacy/*` 别名，确认行为一致。
7. 不带 `Authorization` 和带伪造 `Authorization` 各跑一遍，确认兼容路径不受鉴权影响。
8. 检查 `backend/runtime/legacy-access.log`，确认 gateway 记录旧路径访问。

## 6. 失败判定

- 任一旧路径返回新平台 `code/data` 标准 envelope 且缺少旧字段，判定为不兼容。
- `callback` 不触发、触发时带 OpenAPI HMAC 签名头、或 body 不含 `success/data`，判定为不兼容。
- `language/use_itn/hotwords` 没有进入 ASR 请求，判定为不兼容。
- `legacy.enabled=false` 时除 health 外旧业务路径未返回 410，判定为开关失效。
