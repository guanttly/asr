# 11 — 验收清单

> 用于 v0.3 GA 前的最终走查。每条都需测试用例 + 通过证据（截图 / 录像 / 日志）。

## 1. 功能验收

### 1.1 应用注册与凭证

| 编号 | 项 | 期望 |
|---|---|---|
| ACC-APP-01 | 创建应用 → 仅一次返回 `app_secret` | 二次 GET 返回掩码 |
| ACC-APP-02 | rotate-secret 后旧 token 立即失效 | `ERR_OPEN_AUTH_REVOKED` |
| ACC-APP-03 | disable 应用后所有调用 403 | `ERR_OPEN_APP_DISABLED` |
| ACC-APP-04 | 删除应用 → 软删 / token 立刻失效 / `open_call_logs` 保留 | 数据库行有 `revoked` 标记 |
| ACC-APP-05 | 同名应用拒绝创建 | 409 |

### 1.2 鉴权链路

| 编号 | 项 | 期望 |
|---|---|---|
| ACC-AUTH-01 | 未携带 Authorization 调用任意 `/openapi/v1/*` | 401 `ERR_OPEN_AUTH_MISSING` |
| ACC-AUTH-02 | 错误签名 token | 401 `ERR_OPEN_AUTH_INVALID` |
| ACC-AUTH-03 | 过期 token | 401 `ERR_OPEN_AUTH_EXPIRED` |
| ACC-AUTH-04 | 调用未勾选能力路由 | 403 `ERR_OPEN_CAP_DENIED` |
| ACC-AUTH-05 | 限流：单应用 31 req/s 持续 5 s | 至少 1 条 429 + Retry-After |
| ACC-AUTH-06 | `/api/admin/*` 拒绝 OpenAuth token | 401 |
| ACC-AUTH-07 | `/openapi/v1/*` 拒绝 admin JWT | 401 |

### 1.3 语音转文字

| 编号 | 项 | 期望 |
|---|---|---|
| ACC-ASR-01 | 同步识别 60 s 标准音 | 字错率 ≤ 8%，端到端 ≤ 5 s |
| ACC-ASR-02 | VAD 分段返回 segments 数 ≥ 2 | segments 时间戳单调递增 |
| ACC-ASR-03 | 异步任务：1 GB 音频上传 + 查询 + 取消 | 状态机迁移正确 |
| ACC-ASR-04 | 流式：30 min 持续推送 + 30% 丢包 | 不掉线，自动续接 |
| ACC-ASR-05 | callback 模式：3 次重试后仍失败 → 标记 task 异常 | Dashboard 可见 |
| ACC-ASR-06 | 不支持的格式 / 超大文件 | 415 / 413 |

### 1.4 会议纪要

| 编号 | 项 | 期望 |
|---|---|---|
| ACC-MT-01 | 30 min 录音 audio-summary 同步 | ≤ 15 min 完成 |
| ACC-MT-02 | 标准版部署调用 | 403 `ERR_EDITION_LIMITED` |
| ACC-MT-03 | 非含 meeting_summary 节点的工作流 | 422 `ERR_WORKFLOW_INVALID` |
| ACC-MT-04 | LLM 失败但 ASR 成功 | 422 `ERR_ASR_PARTIAL`，载荷含 `asr` 字段 |
| ACC-MT-05 | text-summary 不含 ASR 字段 | data 中 `asr` 不存在 |
| ACC-MT-06 | regenerate 1 次/分钟限制 | 第 2 次 429 |

### 1.5 Skill 与回调

| 编号 | 项 | 期望 |
|---|---|---|
| ACC-SKL-01 | 注册 skill：callback 探活通过 | 201 |
| ACC-SKL-02 | callback 不可达 | 422 `ERR_SKILL_CALLBACK_UNREACHABLE` |
| ACC-SKL-03 | callback URL 不在白名单 | 422 `ERR_SKILL_CALLBACK_NOT_WHITELISTED` |
| ACC-SKL-04 | dry-run 命中 + 不真正回调 | data.matched=true，三方未收到请求 |
| ACC-SKL-05 | 真实命中：签名校验通过 + 写 `skill_invocations` | 数据库行存在 |
| ACC-SKL-06 | 5 次连续失败 → 自动禁用 + 通知管理员 | skill.enabled=false |
| ACC-SKL-07 | rotate-secret 后旧签名失效 | 三方校验失败 |

### 1.6 工作流附加

| 编号 | 项 | 期望 |
|---|---|---|
| ACC-WF-01 | 不传 workflow_id → 走 default_workflows | 响应 `workflow_origin=app_default` |
| ACC-WF-02 | 应用未配 default → 走 system 默认 | `workflow_origin=system_default` |
| ACC-WF-03 | 显式传不属于本应用的 workflow_id | 404 `ERR_WORKFLOW_NOT_FOUND` |
| ACC-WF-04 | 工作流执行超时 | 422 `ERR_WORKFLOW_TIMEOUT` + partial_data |
| ACC-WF-05 | 删除被某应用引用为 default 的工作流 | 拒绝 + 显式提示 |

### 1.7 兼容接口（完全无鉴权）

| 编号 | 项 | 期望 |
|---|---|---|
| ACC-LEG-01 | **不带任何头**调旧 9 条路径 | 200 / 业务正常，行为与 0.2.x 等价 |
| ACC-LEG-02 | 带伪造 / 错误的 `Authorization` 调旧路径 | 200，token 被忽略，**不**返回 401 |
| ACC-LEG-03 | 别名路径 `/api/legacy/*` 与旧路径等价 | 同 handler，同响应 |
| ACC-LEG-04 | 9 条接口响应字段与 0.2.x 录像逐字段一致 | diff 为空 |
| ACC-LEG-05 | 旧 `audio/to_summary` callback 链路 | 三方收到回调，载荷字段名不变 |
| ACC-LEG-06 | 旧路径传 `workflow_id` | 字段被忽略，无任何日志标记 |
| ACC-LEG-07 | gateway `legacy.enabled=false` | 旧路径与别名同时返回 410 Gone |
| ACC-LEG-08 | legacy 调用**不**写 `open_call_logs` / `skill_invocations` | 表中无对应行 |
| ACC-LEG-09 | legacy 调用**不**写 `runtime/openapi-logs/` | 仅出现在 `runtime/legacy-access.log` |
| ACC-LEG-10 | Dashboard 按 app_id 维度数据不包含 legacy 流量 | 总计数与 `open_call_logs` 一致 |

## 2. 安全验收（仅约束新平台 `/openapi/v1/*`）

| 编号 | 项 | 期望 |
|---|---|---|
| SEC-01 | 端到端 HTTPS（自签证书生效） | TLS 1.2+ 握手成功 |
| SEC-02 | secret 不出现在日志 / 响应 / Dashboard | grep 全部日志为空 |
| SEC-03 | callback URL 阻止指向 RFC1918 / metadata | 注册返回 422 |
| SEC-04 | 重放（同一签名 token 多次调用）允许，但 `?nonce=` 重复仍允许（v0.3 不强制 nonce） | 行为符合规格 |
| SEC-05 | 越权调取另一应用的 meeting_id | 404 |
| SEC-06 | OpenAPI token 不能调用 `/api/admin` | 401 |
| SEC-07 | 兼容接口刻意**不在**安全验收范围 | 见 ACC-LEG-*；`SEC-01 ~ 06` 不针对 `/api/legacy/*` 与旧 9 条路径 |

## 3. 性能验收

|---|---|---|
| PERF-01 | OpenAuth 中间件 P95 增加延迟 | ≤ 5 ms |

## 7. 出口准则

- 第 1-6 节中所有"项"全部通过；
- 试点客户接入并运行 ≥ 7 天；
- 无 P0 / P1 缺陷；P2 缺陷 ≤ 5 个并附绕行方案；
- 项目总结记录归档到 `docs/plans/v0.3/release-notes.md`（GA 当周补）。
