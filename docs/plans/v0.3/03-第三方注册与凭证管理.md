# 03 — 第三方注册与凭证管理

## 1. 注册流程

```
[ 平台管理员 ]                                 [ 三方系统 ]
      │                                              │
      │ 1. 登录管理后台                              │
      │ 2. 创建应用：填名称、备注、勾选能力          │
      │    选择默认工作流（按能力）                  │
      │ 3. 系统返回 app_id + app_secret（仅一次）   │
      │ 4. 安全渠道转交三方                         │ ← 平台不主动外发
      │                                              │
      │                                              │ 5. 在自己服务里保存 app_id / app_secret
      │                                              │ 6. 调用 /openapi/v1/auth/token 拿 access_token
      │                                              │ 7. 调用业务 API 时带 Bearer token
```

只有平台管理员（角色 `admin`）能创建 / 修改 / 删除应用。三方系统不直接拥有自助注册入口（v0.3 范围内）。

## 2. 应用属性

| 字段 | 必填 | 说明 |
|---|---|---|
| `name` | ✅ | 应用名（同租户内唯一） |
| `description` | ❌ | 备注 |
| `allowed_caps[]` | ✅ | 勾选的能力（≥ 1） |
| `default_workflows{}` | ❌ | 每条能力的默认工作流；为空则平台兜底默认 |
| `callback_whitelist[]` | ❌ | 仅当勾选 `skill.invoke` 时必填，限制 skill 回调可用的域名 / IP |
| `rate_limit_per_sec` | ❌ | 默认 30；管理员可调 |
| `secret_version` | 自动 | 每次重置 +1，使旧 token 失效 |

## 3. 管理 API（管理后台用）

> 这些接口属于 `/api/admin/openplatform/...`，沿用现有 JWT 鉴权，仅 `admin` 角色可访问。

| 方法 | 路径 | 用途 |
|---|---|---|
| `GET` | `/api/admin/openplatform/apps` | 列表 + 分页 |
| `POST` | `/api/admin/openplatform/apps` | 创建（响应里**仅此一次**返回 `app_secret`） |
| `GET` | `/api/admin/openplatform/apps/:id` | 详情 |
| `PUT` | `/api/admin/openplatform/apps/:id` | 改名、改备注、改能力清单、改默认工作流 |
| `DELETE` | `/api/admin/openplatform/apps/:id` | 软删除 → status=revoked，token 立即失效 |
| `POST` | `/api/admin/openplatform/apps/:id/rotate-secret` | 重置 secret（响应里返回新 secret 一次） |
| `POST` | `/api/admin/openplatform/apps/:id/disable` | 临时禁用（status=disabled） |
| `POST` | `/api/admin/openplatform/apps/:id/enable` | 启用 |
| `GET` | `/api/admin/openplatform/apps/:id/calls` | 该应用的调用日志（最近 7 天） |
| `GET` | `/api/admin/openplatform/capabilities` | 平台支持的能力清单（前端勾选组件用） |

### 3.1 创建应用响应（示例）

```json
POST /api/admin/openplatform/apps
{
  "name": "客服坐席助手",
  "description": "供客服系统调用语音转文字",
  "allowed_caps": ["asr.recognize", "asr.stream", "meeting.summary"],
  "default_workflows": { "asr.recognize": 12, "meeting.summary": 7 },
  "rate_limit_per_sec": 30
}

200 OK
{
  "app_id": "app_3kj2yA9NqLZmXr",
  "app_secret": "sk_xxxxxxxxxxxxxxxxxxxxxxxxxxxx",   ← 仅此一次返回
  "secret_version": 1,
  "status": "active",
  "allowed_caps": [...],
  "created_at": "2026-04-28T09:12:00Z"
}
```

后续 `GET /api/admin/openplatform/apps/:id` 不会再回显 secret，仅返回掩码 `sk_xxx****xxxx`。

## 4. 三方对接方拿到凭证后的标准动作

```bash
# 1. 用 app_id + app_secret 换 access_token
curl -X POST https://<host>/openapi/v1/auth/token \
  -H "Content-Type: application/json" \
  -d '{"app_id":"app_3kj2yA9NqLZmXr","app_secret":"sk_xxx..."}'
# 200 OK
# { "access_token": "eyJhbGciOi...", "expires_in": 7200, "token_type": "Bearer",
#   "allowed_caps": ["asr.recognize","asr.stream","meeting.summary"] }
```

```bash
# 2. 用 access_token 调用业务 API
curl -X POST https://<host>/openapi/v1/asr/recognize \
  -H "Authorization: Bearer eyJhbGciOi..." \
  -F "file=@meeting.wav" \
  -F "language=zh"
```

## 5. 凭证安全约束

| 约束 | 说明 |
|---|---|
| 明文 secret 仅显示一次 | 平台只存 `Argon2id(app_secret)`；管理员遗失需 rotate |
| access_token TTL ≤ 2 小时 | 默认 7200 秒，可按应用单独调整 |
| token 不可手工撤销 | 通过 rotate-secret 或 disable 应用整体失效 |
| TLS 强制 | gateway 同 v0.2 默认开启 HTTPS；明文 HTTP 仅在内网环境且日志告警 |
| 日志脱敏 | access_token / app_secret / 用户音频 URL 不进入应用日志 |
| 管理员审计 | 创建 / 修改 / 重置 / 禁用 → 写 `admin_audit_log`，含操作者 ID |

## 6. 失败场景与排错

| 现象 | 大概率原因 | 提示 |
|---|---|---|
| 401 ERR_OPEN_AUTH_INVALID | secret 错或 app_id 错 | 让对接方核对 ID；不能直接告诉对方"是 app_id 错还是 secret 错"，统一返回签名无效 |
| 401 ERR_OPEN_AUTH_EXPIRED | access_token 过期 | 重新 `/auth/token` |
| 401 ERR_OPEN_AUTH_REVOKED | rotate 之后还在用旧 token | 让对接方刷新 |
| 403 ERR_OPEN_CAP_DENIED | 调了未勾选的能力 | 后台为该应用补勾能力 |
| 403 ERR_OPEN_APP_DISABLED | 应用被禁用 | 联系管理员 |
| 429 ERR_OPEN_RATE_LIMITED | 超过限流 | 看响应头 Retry-After |

错误码命名前缀统一：`ERR_OPEN_*`，便于对接方做错误处理。
